Understanding GitHub Repositories
Beginner

What is a Repository?

A repository (or repo) is essentially a project's folder where all of the project files are stored, including the revision history of each file.

Key Components of a Repository:

  • README.md - A file that describes your project (uses Markdown formatting)
  • License - A file that defines how others can use your code
  • Code files - The actual source code of your project
  • Commit history - A record of all changes made to the repository

Creating a Repository

To create a new repository on GitHub:

  1. Click the '+' icon in the top right and select "New repository"
  2. Name your repository
  3. Add a description (optional but recommended)
  4. Choose between public (visible to everyone) or private (visible only to you and collaborators)
  5. Initialize with a README (good practice for new projects)
  6. Click "Create repository"

Repository Structure Example:

my-project/
│
├── README.md
├── LICENSE
├── .gitignore
├── src/
│   └── index.js
├── docs/
│   └── instructions.md
└── tests/
    └── test.js

Hint

Remember that a README file is meant to help others understand your project. It's typically the first file someone will look at when they visit your repository.

Repository names should be URL-friendly, which is why lowercase and hyphens are recommended.

The .gitignore file helps keep your repository clean by excluding files that don't need to be tracked, like temporary files or build outputs.
